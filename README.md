# Auto24 — CRM-система записи на занятия для автошколы

Telegram-бот для учеников + веб-панели для администратора и инструкторов.

## Что нового в этой версии

1. **Инструкторы** — теперь можно добавлять через панель администратора: ФИО, фото, модель авто, коробка передач, годы стажа, отделение.
2. **Реальная авторизация** — вместо ID+имя теперь пароль у админа и у каждого инструктора. Сессия хранится в токене (`Authorization: Bearer ...`), токен живёт в `localStorage` на фронте — **обновление страницы больше не разлогинивает**.
3. **Отвязка Telegram у ученика** — если ученик сменил аккаунт/телефон и бот его не узнаёт, админ жмёт «Отвязать Telegram» в панели, ученик заново вводит свой ключ в боте. Плюс бот теперь ловит конфликт (если Telegram уже привязан к другому ученику) и сам просит обратиться к администратору вместо технической ошибки.
4. **Отделения (филиалы)** — админ может добавлять/удалять отделения и привязывать к ним инструкторов.
5. **Фронт переделан с нуля по архитектуре**: больше не один HTML-файл на 500 строк, а разделение на `frontend/` (страницы), `static/css/` и `static/js/` (стили и логика по файлам, каждая вкладка — свой JS-файл).

## Запуск

1. Установи зависимости:
```
pip install -r requirements.txt
```

2. Скопируй `.env.example` в `.env`, впиши токен бота и данные PostgreSQL.

3. Заполни базу тестовыми данными:
```
python seed.py
```
В консоли выведет:
- тестовый ключ ученика для бота
- пароль сид-инструкторов (`instructor2024`, ID 1–4)

Пароль администратора смотри/меняй в `routers/auth_simple.py` → `ADMIN_PASSWORD` (по умолчанию `auto24-admin-2026`, обязательно смени перед реальным деплоем и вынеси в `.env`).

4. Запусти сервер:
```
uvicorn main:app --reload
```

5. Запусти бота отдельным процессом:
```
python bot.py
```

6. Открой панели в браузере:
- Вход (общий): `http://127.0.0.1:8000/app/login.html`
- После входа само перекинет на `/app/admin/dashboard.html` или `/app/instructor/dashboard.html`

## Структура проекта

```
main.py                  — сборка FastAPI-приложения, подключение роутеров и статики
database.py              — подключение к БД
models.py                — модели SQLAlchemy
schemas.py                — Pydantic-схемы ответов API
seed.py                   — наполнение базы тестовыми данными

routers/
  auth_simple.py          — сессии, токены, require_admin / require_instructor
  auth.py                 — /auth/me — фронт им проверяет, жив ли токен из localStorage
  admin.py                 — заявки/расписание инструктора + управление учениками (только админ)
  instructors.py           — CRUD инструкторов + вход инструктора
  branches.py               — CRUD отделений
  students.py                — идентификация ученика в боте, статус, привязка Telegram
  slots.py                    — публичные эндпоинты для бота (список свободных слотов и т.п.)

static/
  css/style.css            — общий дизайн админ-панели и логина
  css/instructor.css        — дизайн панели инструктора (мобильный, отдельный стиль)
  js/auth.js                 — общий хелпер: токен в localStorage, apiFetch(), requireRole()
  js/admin.js                 — переключение вкладок в панели администратора
  js/admin-students.js         — вкладка "Ученики"
  js/admin-instructors.js       — вкладка "Инструкторы"
  js/admin-branches.js           — вкладка "Отделения"
  js/instructor.js                — вся логика панели инструктора (заявки, расписание, настройки)

frontend/
  login.html                — единая страница входа (переключатель Админ/Инструктор)
  admin/dashboard.html       — панель администратора (вкладки: Ученики/Инструкторы/Отделения)
  instructor/dashboard.html   — панель инструктора (Заявки/Расписание/Настройки)

photos/                      — фото инструкторов, раздаются по /photos/...
bot.py, keyboards.py, states.py — Telegram-бот (aiogram), без изменений в логике, кроме
                                   обработки конфликта telegram_id в process_key()
```

## Как устроена авторизация (коротко)

- `POST /admin/login` (пароль) → токен, роль `admin`
- `POST /instructors/login` (ID + пароль) → токен, роль `instructor`, `instructor_id`
- Все защищённые эндпоинты ждут заголовок `Authorization: Bearer <токен>`
- Токены хранятся в памяти сервера (`SESSIONS` в `auth_simple.py`) — при перезапуске сервера все сессии слетают, это нормально для MVP. Если понадобится переживать перезапуск — вынести `SESSIONS` в таблицу в БД.
- На фронте токен лежит в `localStorage`, поэтому F5 не разлогинивает — это и была изначальная проблема.

## Деплой на несколько поддоменов на одном сервере

Идея с "витриной проектов" технически реализуется через nginx: один сервер (например, на Beget или Timeweb), несколько systemd-сервисов (каждый проект — свой `uvicorn` процесс на своём порту, например Auto24 на 8001, CompShop на 8002), и nginx как reverse proxy, который по `server_name auto24.твой-домен.ru` проксирует на `127.0.0.1:8001`, а `server_name compshop.твой-домен.ru` — на `127.0.0.1:8002`. Это отдельная тема для следующего захода — если понадобится, распишем конфиг nginx и systemd-юниты пошагово.
